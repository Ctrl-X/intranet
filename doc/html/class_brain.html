Documentation des fonctions membres

beforeLoad (   $request = null)

Lance l'exécution de scripts avant le chargement de la page.

Vérifie si la page visité nécessite d'être connecté ou des paramètres ($_GET / $_POST)

Paramètres
$requestUn tableau du type array('log' => array('request' => true, 'redirect' => 'index.php'), 'params' => array('request' => array('MODULE', 'ACTION'), 'redirect' => 'index.php'))
Renvoie
ne retourne rien, stop le chargement si le paramètre n'est pas passé
getDetailledModule ( )

Appel la version détaillé d'un module.

Récupère un module si des paramètres existent (ACTION, MODULE, CONTEXT) et lui passe l'ensemble des paramètres ($_GET / $_POST).

Renvoie
inclusion du module via getModule()
getModule (   $_COMMAND = null)

Récupère un module.

Appel un module en lui passant des commandes soit l'ACTION, le CONTEXT, ... Le module Brain gère ses actions à ce niveau (installation de module, ...)

Paramètres
$_COMMAND= array()
Renvoie
inclut un fichier php
installModules ( )

Lance l'installation des nouveaux modules.

Vérifie quels sont les modules installés puis regarde dans le dossier 'modules' ceux qui ne le sont pas et les installe si le fichier d'installation existe (un fichier .php qui porte exactemement le même nom que son dossier).

Renvoie
ne retourne rien, affiche un message lorsqu'un fichier est manquant ou que le module fait partie de la blacklist
printModule (   $params = array( 'position' => 'normal', 'context' => 'tile', 'limit' => false))

Affiche un ensemble de module.

Sélectionne l'ensemble des modules dans la base de donnée selon les paramètres passés puis les appel via getModule()

Paramètres
$params= array()
'position'conditionne la récupération des modules à la position qu'ils doivent avoir (modules ne s'affichant que sur l'accueil, modules ne s'affichant que sur la page profil, ...)
'context'fait partie des informations dont nécessite le module et peut être de différent type (tile, detail, ...). L'affichage varie selon le contexte
'limit'pour donner une limite à la récupération de ce module (les 5 derniers, ...)
Renvoie
appel la fonction getModule() autant de fois que de modules demandés
